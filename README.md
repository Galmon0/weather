# Погода России — карта и прогноз

Сбор погоды по областным городам России в Postgres (раз в день) + карта и
анализ во Streamlit + ML-прогноз температуры на завтра.

## Стек
- **Парсер** meteo9.ru → pandas — `weather/source_meteo9.py`
- **Postgres** (Docker) — хранилище (`db/schema.sql`)
- **ETL + APScheduler** — загрузка раз в день (`weather/load.py`, `weather/scheduler.py`)
- **Streamlit + Plotly** — карта и графики (`app/streamlit_app.py`)
- **scikit-learn** — прогноз на завтра *(в работе)*

## Запуск
1. Установить **Docker Desktop**.
2. (необязательно) `copy .env.example .env` и поправить время загрузки.
3. `docker compose up -d --build`
4. Открыть приложение: http://localhost:8501
5. Загрузить данные сразу, не дожидаясь расписания:
   `docker compose run --rm loader python -m weather.load`

Сервис `loader` и так делает стартовую загрузку при `up`, а дальше — ежедневно
по расписанию (`SCHEDULE_HOUR`/`SCHEDULE_MINUTE`).

## Локальная проверка парсера (без Docker)
`py -m weather.source_meteo9`

## Каталог городов
`weather/stations.csv` — код станции meteo9, координаты, таймзона (10 городов в MVP).
Добавить город: `resolve_city("Название")` из `source_meteo9` подскажет код станции.

## Известные ограничения
- `robots.txt` сайта запрещает `/responses/`. Используем бережно: раз в день,
  с кэшированием в БД (не перезапрашиваем загруженное) и задержками. Источник
  сделан сменным (`WEATHER_SOURCE`) — задел под переход на Open-Meteo.
- **Осадки** на источнике зашумлены (коды 65535 и 990–999) — поле ненадёжное,
  значения ≥900 отбрасываются. Температура/давление/влажность/ветер — чистые.
- Время наблюдений хранится как отдаёт источник (UTC/местное — уточняется);
  в каталоге есть `tz_offset` для последующей нормализации.
